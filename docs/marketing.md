# X・note 発信連携設計

## 最終更新：2026-06-19

---

## 基本戦略：「完成品」ではなく「作る過程」を売る

既存の金融教育コンテンツは完成品で勝負する。
個人が勝てるのは **制作プロセスの公開**。Claude Code × 子育て × 金融教育という掛け算は希少で、共感とノウハウ両方が発信ネタになる。
→ プロダクトが完成する前から発信を始められる＝認知の先行投資ができる。

---

## 発信の3軸（PPC）

| 軸 | 内容 | 主な媒体 |
|---|---|---|
| **P**roduct | money-quest そのもの・遊んでみてネタ | X（短）→ ゲームURL |
| **P**rocess | Claude Codeでどう作ったか・詰まった所 | X（連投）→ note（詳細） |
| **C**hild | 自分の10歳の子のリアルな反応・学び | X（共感）→ note（体験談） |

---

## 「作る過程」のコンテンツ化ネタ（ストック）

制作中に自然に出てくる発信ネタ：
1. 「子どもの金融教育、ゲームで作ることにした」（起点・宣言）
2. 「Claude Codeにアイデア吐き出したらここまで形になった」（過程・驚き）
3. 「複利の概念を"魔法"に翻訳してみた」（設計の工夫）
4. 「詐欺ダンジョン作ってたら自分が一番勉強になった」（大人の学び）
5. 「10歳の息子にテストプレイさせた結果」（子の反応・最強ネタ）
6. 「金融庁の4分野をRPGの4エリアに変換した話」（設計の裏側・信頼性）
7. 「無料で公開します」（完成・拡散点）

→ 各ネタはXで反応を見て、伸びたものをnote記事に深掘りして昇華する。

---

## 発信の順番（タイムライン）

```
【承認前・制作中】
X：宣言→過程を小出し（毎回の進捗をネタ化）
    ↓ フォロワー・期待を貯める
note：制作の裏側を1〜2本（信頼構築）

【MVP完成】
X：「エリア1できた、遊んでみて」＋ゲームURL
    ↓ 反応・フィードバックもネタに
note：「Claude Codeで金融教育ゲームを作る」体験談（本命記事）

【全体完成・ASP承認後】
X：完成告知＋子どもの反応で拡散狙い
note：完成版＋親向け解説（一部有料）
    ↓ ゲーム内導線と相互送客
investment-quiz / アフィリエイトへ
```

---

## 媒体ごとの役割分担

| 媒体 | 役割 | ゴール |
|---|---|---|
| X | 認知・拡散・反応収集 | プロフィール→note/ゲームへ誘導 |
| note | 信頼構築・体験談ストック・有料化 | 読者を顧客化、ストック収益 |
| money-quest | 体験・エンゲージ | 大人向け導線へ送客 |
| investment-quiz | 自分ごと化・診断 | アフィリエイト着地 |

**相互送客の輪：X→note→ゲーム→診断→アフィリエイト→（実績をまたXで発信）**
